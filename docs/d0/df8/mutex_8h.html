◆ ls_mutex_lock()

ls_result_t ls_mutex_lock ( ls_mutex_t *const  mutex)

Locks a mutex.

If it is already locked, it will wait indefinitely until it is unlocked.

Parameters
mutexThe mutex to lock.
Returns
LS_E_NULL if mutex is NULL;
LS_E_MAGIC if the struct magic is not set;
LS_E_UNINITIALIZED if mutex->lock is NULL (Windows);
LS_E_SUCCESS on success;
LS_E_ABANDONED if the lock was abandoned;
LS_E_FAILURE on failure;
LS_E_UNSUPPORTED if the implementation is missing.

ls_result_t ls_mutex_timedlock ( ls_mutex_t *const  mutex,
const struct timespec  timeout 
)

Locks a mutex.

Same as ls_mutex_lock, except that this function will time-out after the specified timeout while waiting.

Parameters
mutexThe mutex to lock.
timeoutThe time to wait before timing out.
Returns
LS_E_INVALID if a timeout of zero was given;
LS_E_CONVERSION if conversion of timeout failed;
LS_E_TIMEOUT if it times out;
return values from ls_mutex_lock otherwise.
